# Happen 线框规格

本文定义 Happen MVP 的结构型线框规格，用于 Figma Wireframe 制作。

本文不定义视觉风格、颜色、字体、品牌资产、插画、动效或精细 UI 表现。

## App Structure

Happen 是一个以微信小程序为优先平台的儿童健康事实记录与回顾工具。MVP 应该像一个快速记录工作台，而不是一个宽泛的记录管理应用。

应用始终围绕一个当前选中的孩子展开。每条记录、每个回顾页面、每个表单和每个设置项都属于当前孩子上下文。

主要结构：

1. 首次使用设置
   - 仅在用户没有可访问孩子时出现。
   - 创建最小可用上下文：孩子昵称和照护者标签。
2. 首页
   - 设置完成或打开应用后的默认工作台。
   - 展示当前孩子、排便状态、快速记录动作和回顾入口。
3. 孩子设置
   - 管理最小孩子信息和照护者信息。
   - 包含邀请照护者。
4. 记录
   - 从首页进入的二级纠错和浏览页面。
   - 不是主导航目的地。
5. 生病时间线
   - 用于就诊前后按日期范围回顾的页面。
6. 记录表单和面板
   - 排便确认 Sheet。
   - 排便记录编辑器。
   - 生病日记录表单。
7. 邀请加入流程
   - 通过微信分享卡片或邀请链接打开的单孩子共享访问流程。

信息层级：

1. 快速记录已经发生的事实。
2. 立即回顾距上次排便的间隔。
3. 为就诊沟通回顾生病时间线。
4. 记录纠错和浏览。
5. 孩子与照护者设置。

范围边界：

- MVP 只记录事实；不提供诊断、医疗建议、用药推荐、剂量计算、图表、自动识别生病周期、社交功能、聊天、评论或家庭角色管理。
- 使用具体语言，例如“记录排便”“记录生病情况”“生病时间线”。不要把“事件”作为主要用户概念。

## Navigation

MVP 应采用首页优先的导航模型。

推荐导航：

- 打开应用时，如果用户没有孩子，进入首次使用设置。
- 打开应用时，如果用户至少有一个可访问孩子，进入首页。
- 首页是主要工作台和主要返回点。
- 孩子设置从首页的小入口进入。
- 记录从首页的次级入口进入。
- 生病时间线从首页可见的回顾动作进入。
- 记录表单根据用户意图从首页、时间线或记录页打开。
- 邀请确认从外部邀请链接或微信分享卡片打开。

底部导航：

- MVP 优先不使用底部 Tab。
- 如果实现必须使用 Tab，仅使用首页和设置。
- 不要把记录放入底部 Tab。

孩子上下文导航：

- 如果用户只有一个孩子，只展示孩子昵称作为上下文，不增加选择摩擦。
- 如果用户有多个孩子，点击孩子昵称打开孩子选择器。
- 切换孩子后，首页状态、记录、生病时间线、表单和设置都更新。
- 表单在保存或确认前必须展示当前孩子。
- 如果 MVP 包含添加另一个孩子，应放在孩子设置中，而不是作为孩子选择器的主路径。

返回和关闭行为：

- 关闭 Sheet 返回上一屏，不保存更改。
- 表单已有输入内容时，返回才提示是否放弃。
- 快速排便记录成功后直接回到首页。
- 生病日保存成功后回到上一屏。
- 时间线日期范围变更停留在时间线。
- 记录页返回首页。
- 无效或过期邀请状态不能通过该邀请进入首页。

## Screen Inventory

### 1. 首次使用设置

目的：为新用户创建最小孩子上下文。

入口：

- 用户没有可访问孩子时打开应用。

出口：

- 设置成功后进入新建孩子的首页。

必要内容：

- 设置标题。
- 孩子昵称输入框。
- 照护者标签快捷选项。
- 仅在选择自定义时展示自定义照护者标签输入框。
- 主确认动作。

状态：

- 必填项为空。
- 提交失败，保留已输入内容。

排除：

- 不要生日、性别、头像、孩子真实姓名、照护者真实姓名、家庭名称、医疗档案、邀请提示或多孩子提示。

### 2. 首页

目的：作为按孩子限定范围的快速记录工作台。

入口：

- 老用户打开应用。
- 设置完成后。
- 邀请加入后。
- 快速记录成功后。

必要内容：

- 当前孩子昵称。
- 多孩子时的切换入口。
- 设置入口。
- 排便状态。
- 主动作：记录排便。
- 主动作：记录生病情况。
- 回顾动作：生病时间线。
- 次级动作：查看或编辑记录。

状态：

- 暂无排便记录。
- 排便状态加载失败并可重试。
- 当前选择的孩子不可用。
- 多照护者场景下，在有用时展示最后编辑者提示。

排除：

- 不要图表、分析仪表盘、教育内容、引导轮播、邀请提示、社交动态或作为首页主体的通用事件列表。

### 3. 孩子选择器

目的：安全切换当前孩子。

入口：

- 多个孩子可访问时，在首页或其他孩子上下文页面点击孩子昵称。

必要内容：

- 可访问孩子列表。
- 当前孩子标识。
- 每个孩子的点击目标。

行为：

- 用户选择孩子后才切换。
- 切换后，如果上一屏对新孩子仍有效，返回上一屏。
- 所有可见的孩子范围数据刷新为新孩子。

排除：

- 不要把添加孩子做成选择器中的突出动作。
- 不要家庭工作区、家庭分组或角色层级。

### 4. 孩子设置

目的：管理最小孩子信息和照护者信息。

入口：

- 从首页的设置入口进入。

必要内容：

- 孩子昵称字段。
- 当前用户的照护者标签字段。
- 按展示标签显示的照护者列表。
- 邀请照护者动作。
- 仅在 MVP 支持时加入添加另一个孩子动作。

状态：

- 昵称更新失败。
- 照护者标签更新失败。
- 照护者列表加载失败。

排除：

- 不要只读开关、管理员角色、审批流程、聊天、评论、家庭动态或活动流。

### 5. 邀请确认

目的：让被邀请照护者加入一个孩子的共享记录。

入口：

- 有效邀请链接或微信分享卡片。

必要内容：

- 孩子昵称。
- 简短说明：加入后可以查看和编辑这个孩子的 MVP 健康事实。
- 照护者标签快捷选项。
- 需要时展示自定义照护者标签输入框。
- 加入动作。
- 取消或离开入口。

状态：

- 无效邀请。
- 过期邀请。
- 已加入。
- 加入提交失败，保留已选择标签。

排除：

- 不要家庭角色选择、权限矩阵、手机号要求、真实姓名要求或宽泛家庭账号设置。

### 6. 排便确认 Sheet

目的：以最小摩擦显式记录一次排便。

入口：

- 从首页点击记录排便。

必要内容：

- 当前孩子标签。
- 日期字段，默认今天。
- 数量影响文案，说明确认会为所选日期增加 1 次。
- 确认动作。
- 添加详情动作。
- 取消动作。

展开详情内容：

- 可选观察标签。
- 可选短备注。

状态：

- 确认前修改日期。
- 详情已展开。
- 保存失败，保留用户输入。

排除：

- 打开时不自动创建记录。
- 不要大便分类、照片、颜色、量、医疗警报或趋势分析。

### 7. 排便记录编辑器

目的：纠正或补充日期级排便记录。

入口：

- 从记录页点击某个排便日期。
- 如果有暴露，也可从首页纠错路径进入。

必要内容：

- 当前孩子标签。
- 日期字段。
- 数量步进器和直接数字输入。
- 可选观察标签。
- 可选短备注。
- 有用时展示最后编辑者标签。
- 保存动作。
- 删除动作。

状态：

- 数量为空或为零。
- 删除确认。
- 保存失败，保留未保存输入。

排除：

- 不要把此编辑器作为正常快速记录路径。

### 8. 生病日记录表单

目的：创建或更新一个日期级生病记录。

入口：

- 从首页点击记录生病情况。
- 从时间线点击日期行。
- 从记录页点击生病日期。

必要内容：

- 当前孩子标签。
- 从首页打开时日期默认今天。
- 最高体温输入。
- 症状快捷词。
- 症状或备注自由文本字段。
- 用药输入区域。
- 保存动作。

次级内容：

- 如有，可展示该孩子最近输入过的用药词。
- 用药快捷词的“仅用于记录”安全文案。
- 需要时的更多备注空间。
- 有用时展示最后编辑者标签。

状态：

- 所选日期已有记录时以编辑模式加载。
- 日期改到另一个已有记录的日期。
- 仅有日期时尝试保存。
- 返回时有未保存内容。
- 保存失败，保留未保存输入。

排除：

- 不要药品数据库、剂量计算器、用药说明、症状严重度评分、医疗推荐或诊断语言。

### 9. 生病时间线

目的：按日期范围扫描每天的生病事实。

入口：

- 从首页点击生病时间线。

必要内容：

- 当前孩子标签。
- 日期范围选择器。
- 天数。
- 按时间顺序展示有记录的日期。
- 每日条目只展示存在的事实：
  - 最高体温。
  - 用药。
  - 症状。
  - 备注。
  - 有用时展示最后编辑者。
- 对无记录日期做紧凑提示。

状态：

- 默认最近 7 天，结束日期为今天。
- 已调整的有效日期范围。
- 开始日期晚于结束日期。
- 范围内为空，提供记录生病情况或调整日期范围动作。
- 时间线加载失败并可重试。

排除：

- 不要诊断报告布局。
- 不要自动识别生病周期的语言。
- 不要用摘要替代原始用户记录事实。

### 10. 记录

目的：浏览和纠正日期级记录。

入口：

- 从首页点击查看或编辑记录。

必要内容：

- 当前孩子标签。
- 排便 / 生病日分段控件。
- 当前类型的按日期分组列表。
- 当前类型的空状态。
- 当前类型的可选轻量新增动作。

行为：

- 排便日期打开排便记录编辑器。
- 生病日期打开生病日记录表单。
- 记录页是次级页面，不应与首页记录动作竞争。

排除：

- 不要通用事件时间线。
- 不要分析仪表盘。

## Screen Layout

### 首次使用设置布局

顶部区域：

- 简短设置标题。

主体区域：

- 孩子昵称输入。
- 照护者标签快捷选项。
- 条件展示的自定义标签字段。

底部区域：

- 主确认动作。

线框结构：

```text
[设置标题]

[孩子昵称输入]

[照护者标签选项]
[自定义标签输入，如已选择]

[开始 / 确认]
```

### 首页布局

顶部栏：

- 当前孩子昵称放在左侧或中央上下文位置。
- 多孩子时切换入口附着在昵称上。
- 设置入口放在另一侧。

主要状态：

- 排便状态区块，给出直接间隔答案。
- 有记录时展示上次排便日期。
- 有意义时展示今天次数。

主动作：

- 记录排便。
- 记录生病情况。

回顾区域：

- 生病时间线。
- 查看或编辑记录作为次级入口。

线框结构：

```text
[当前孩子]                         [设置]

[排便状态]
[距上次排便天数]
[上次日期]
[今天次数]

[记录排便]
[记录生病情况]

[生病时间线]
[查看或编辑记录]
```

### 孩子选择器布局

主体区域：

- 可访问孩子列表。
- 当前孩子标识。

线框结构：

```text
[选择孩子]

[当前孩子项]
[其他孩子项]
[其他孩子项]
```

### 孩子设置布局

顶部区域：

- 页面标题。

主体区域：

- 孩子昵称可编辑字段。
- 我的照护者标签可编辑字段。
- 照护者列表。

动作区域：

- 邀请照护者。
- 如支持，添加另一个孩子。

线框结构：

```text
[孩子设置]

[孩子昵称字段]
[我的照护者标签字段]

[照护者列表]

[邀请照护者]
[添加另一个孩子，如支持]
```

### 邀请确认布局

顶部区域：

- 邀请状态或加入标题。
- 孩子昵称。

主体区域：

- 共享访问说明。
- 照护者标签选项。
- 条件展示的自定义标签字段。

底部区域：

- 加入动作。
- 取消或离开动作。

线框结构：

```text
[加入孩子记录]
[孩子昵称]

[访问说明]

[照护者标签选项]
[自定义标签输入，如已选择]

[加入]
[取消]
```

### 排便确认 Sheet 布局

头部：

- Sheet 标题。
- 当前孩子标签。

主体区域：

- 日期选择器。
- 数量影响文案。
- 默认折叠的可选详情区域。

动作：

- 确认。
- 添加详情。
- 取消。

线框结构：

```text
[记录排便]
[孩子标签]
[日期选择器]

[将为此日期增加 1 次]

[确认]
[添加详情] [取消]
```

展开详情结构：

```text
[观察标签]
[短备注]
```

### 排便记录编辑器布局

头部：

- 标题和孩子标签。

主体区域：

- 日期选择器。
- 数量控件。
- 观察标签。
- 备注字段。
- 有用时展示最后编辑者。

动作：

- 保存。
- 删除。

线框结构：

```text
[排便记录]
[孩子标签]
[日期选择器]

[数量步进器 + 数值]

[观察标签]
[备注字段]
[最后编辑者，如有用]

[保存] [删除]
```

### 生病日记录表单布局

头部：

- 标题和孩子标签。

主体区域：

- 日期选择器。
- 最高体温输入。
- 症状快捷词。
- 症状或备注文本字段。
- 用药区域。
- 如展开或需要，展示额外备注。
- 有用时展示最后编辑者。

动作：

- 保存。

线框结构：

```text
[记录生病情况]
[孩子标签]
[日期选择器]

[最高体温输入]

[症状快捷词]
[症状 / 备注文本字段]

[用药区域]
[展示用药词时的“仅用于记录”安全文案]

[保存]
```

### 生病时间线布局

顶部区域：

- 标题和当前孩子标签。
- 日期范围选择器。
- 天数。

主体区域：

- 有记录日期按时间顺序优先展示。
- 每日只展示存在的事实。
- 无记录日期做紧凑提示。

空状态：

- 暂无记录提示。
- 记录生病情况动作。
- 调整日期范围动作。

线框结构：

```text
[生病时间线]                 [孩子标签]
[日期范围选择器]
[天数]

[日期]
[体温，如存在]
[症状，如存在]
[用药，如存在]
[备注，如存在]
[最后编辑者，如有用]

[紧凑无记录汇总]
```

### 记录页布局

顶部区域：

- 标题和当前孩子标签。
- 分段控件。

主体区域：

- 按日期分组列表。
- 行摘要。
- 当前类型空状态。

线框结构：

```text
[记录]                         [孩子标签]
[排便 | 生病日]

[日期分组]
[记录摘要行 >]

[日期分组]
[记录摘要行 >]
```

## Components

### 孩子上下文头部

用于首页、时间线、记录、设置和表单。

包含：

- 当前孩子昵称。
- 多孩子时的切换入口。
- 表单上的可选紧凑孩子标签。

规则：

- 保存或确认前必须能看到所选孩子。
- 不要把孩子上下文藏在记录表单的菜单里。

### 孩子选择器

包含：

- 可访问孩子列表。
- 当前孩子标识。

规则：

- 选择会更改当前孩子。
- 不包含家庭管理复杂度。

### 排便状态区块

包含：

- 有数据时展示距上次排便天数。
- 上次日期。
- 有意义时展示今天次数。
- 无记录时展示空状态。

规则：

- 无数据时不要展示错误间隔。
- 状态应是直接答案，而不是图表。

### 主记录动作

包含：

- 记录排便。
- 记录生病情况。

规则：

- 两个动作都必须在首页可见。
- 记录排便应是最容易触达的动作。
- 不要把这些动作藏在记录页后面。

### 回顾动作

包含：

- 生病时间线。
- 查看或编辑记录。

规则：

- 时间线是为就诊沟通服务的回顾界面。
- 记录页是次级纠错和浏览路径。

### 日期选择器

用于：

- 排便确认 Sheet。
- 排便记录编辑器。
- 生病日记录表单。
- 生病时间线。

规则：

- 记录表单日期默认今天。
- 时间线默认最近 7 天，结束日期为今天。
- 时间线开始日期晚于结束日期时必须阻止应用。

### 排便确认 Sheet

包含：

- 孩子标签。
- 日期选择器。
- 数量影响文案。
- 确认。
- 添加详情。
- 取消。
- 展开后的可选标签和备注。

规则：

- 打开 Sheet 不创建或更新记录。
- 确认后为所选孩子和日期增加 1 次。

### 观察标签

建议标签：

- 稀。
- 硬。
- 量少。
- 频繁。
- 用力。

规则：

- 标签是可选快速记录词。
- 标签不是医学分类或异常警报。

### 数量控件

用于排便记录编辑器。

包含：

- 步进器。
- 直接数字输入。

规则：

- 保存时数量必须为正数。
- 为空或为零时应阻止保存，或引导删除。

### 生病日表单字段

包含：

- 日期。
- 最高体温。
- 症状快捷词。
- 症状或备注自由文本。
- 用药文本输入。
- 可选备注扩展。

规则：

- 保存要求至少有一个日期以外的事实。
- 除日期外，各事实字段都单独可选。

### 症状快捷词

可选示例：

- 发热。
- 咳嗽。
- 流鼻涕。
- 呕吐。
- 腹泻。
- 嗓子疼。
- 胃口差。
- 睡得不好。

规则：

- 快捷词添加的是可编辑、可移除的用户记录内容。
- 不触发严重程度评分或建议。

### 用药输入

包含：

- 用药自由文本字段。
- 该孩子最近用过的用药词，如有。
- 展示用药词时的安全文案。

规则：

- 用药词表示已经使用过的药或护理项。
- 不得表现为推荐。
- 如果没有用药历史，优先展示普通文本输入，而不是醒目的通用药名列表。

### 时间线日期条目

包含：

- 日期。
- 只展示存在的字段：体温、用药、症状、备注。
- 有用时展示最后编辑者。
- 点击行打开该日期的生病日记录表单。

规则：

- 省略空字段。
- 不诊断、不做系统判断式摘要、不推断生病周期边界。

### 记录列表

包含：

- 分段类型选择器。
- 日期分组。
- 记录摘要行。

规则：

- 点击行打开对应编辑器。
- 记录页用于浏览和纠错。

### 照护者列表

包含：

- 有该孩子访问权限的照护者展示标签。

规则：

- 允许重复标签。
- 标签不暗示权限级别。

### 邀请加入模块

包含：

- 孩子昵称。
- 访问说明。
- 照护者标签选项。
- 加入动作。

规则：

- 邀请限定到一个孩子。
- 只有显式确认加入后才创建成员关系。
- 无效或过期邀请不创建成员关系。

### 错误和空状态

可复用状态：

- 无孩子。
- 无排便记录。
- 所选范围无生病记录。
- 加载失败并可重试。
- 保存失败且保留输入。
- 无效邀请。
- 过期邀请。
- 所选孩子不可用。

规则：

- 空状态应平静、事实化。
- 错误状态不能以副作用创建、编辑或删除记录。

## User Flows

### 流程 1：首次使用

1. 用户打开 Happen。
2. 系统检查可访问孩子。
3. 如果没有孩子，展示首次使用设置。
4. 用户输入孩子昵称。
5. 用户选择或输入照护者标签。
6. 用户确认。
7. 系统创建孩子和成员关系。
8. 用户进入该孩子首页。

线框要求：

- 单屏设置。
- 孩子昵称和照护者标签存在后才允许确认，或确认时阻止。
- 设置期间不创建健康事实。

### 流程 2：老用户打开首页

1. 用户打开 Happen。
2. 系统加载上次选择的可访问孩子，或选择唯一孩子。
3. 首页展示孩子上下文和排便状态。
4. 用户可以立即记录排便或生病情况。

线框要求：

- 首页记录前不需要额外导航。
- 排便状态可以展示加载、空或已加载状态。

### 流程 3：切换孩子

1. 用户点击当前孩子昵称。
2. 如果有多个可访问孩子，打开孩子选择器。
3. 用户选择一个孩子。
4. 上一屏以新的孩子上下文重新加载。
5. 之后的保存或确认动作只作用于新选中的孩子。

线框要求：

- 当前孩子标识可见。
- 表单在日期或保存上下文附近展示当前孩子。

### 流程 4：记录一次排便

1. 用户打开首页。
2. 用户点击记录排便。
3. 确认 Sheet 打开，展示当前孩子和今天日期。
4. 用户确认，或修改日期后确认。
5. 系统为该孩子和日期的次数增加 1。
6. 用户返回首页。
7. 首页更新间隔和今天次数。

线框要求：

- 快速路径是：首页 -> 记录排便 -> 确认。
- 添加详情可选且次要。
- 打开 Sheet 不创建记录。

### 流程 5：快速记录时补充排便详情

1. 用户打开排便确认 Sheet。
2. 用户点击添加详情。
3. 标签和短备注在 Sheet 内展开。
4. 用户选择标签或输入备注。
5. 用户确认。
6. 系统保存该日期的增加次数和详情。

线框要求：

- 详情不跳转到完整编辑器。
- 确认仍是主要完成动作。

### 流程 6：纠正排便记录

1. 用户从首页打开记录。
2. 用户选择排便分段。
3. 用户点击某个日期行。
4. 打开排便记录编辑器。
5. 用户编辑日期、数量、标签或备注。
6. 用户保存或删除。
7. 系统更新记录并重新计算首页状态。

线框要求：

- 数量可以直接纠正。
- 删除需要确认。
- 零或空数量不能作为普通记录保存。

### 流程 7：记录生病日

1. 用户打开首页。
2. 用户点击记录生病情况。
3. 生病日记录表单打开，展示当前孩子和今天日期。
4. 用户输入至少一个日期以外的事实。
5. 用户保存。
6. 系统为该孩子和日期创建或更新记录。
7. 用户返回上一屏。

线框要求：

- 仅有日期时阻止保存。
- 表单支持一个快速事实加保存。
- 展示用药快捷词时包含“仅用于记录”的安全文案。

### 流程 8：更新已有生病日

1. 用户打开时间线或记录页。
2. 用户点击某个日期。
3. 如果该日期有记录，生病日记录表单以编辑模式打开。
4. 如果没有记录，表单以该日期的新建模式打开。
5. 用户编辑并保存。

线框要求：

- 已有记录被编辑，而不是重复创建。
- 日期改到另一个已有记录的日期时，不得静默覆盖任一记录。

### 流程 9：就诊时间线回顾

1. 用户打开所选孩子首页。
2. 用户点击生病时间线。
3. 时间线默认打开最近 7 天，结束日期为今天。
4. 用户扫描有记录日期。
5. 用户按需调整日期范围。
6. 用户使用按日事实解释近期生病情况。

线框要求：

- 日期范围和天数可见。
- 有记录日期易于扫描。
- 空日期紧凑呈现。
- 页面不要像诊断输出。

### 流程 10：邀请照护者

1. 已有照护者打开孩子设置。
2. 已有照护者点击邀请照护者。
3. 系统提供按孩子限定范围的邀请链接或微信分享卡片。
4. 已有照护者在日常记录路径之外分享。

线框要求：

- 邀请动作位于设置中。
- 邀请清楚限定到当前孩子。

### 流程 11：被邀请照护者加入

1. 被邀请用户打开邀请。
2. 系统校验邀请。
3. 有效邀请展示邀请确认。
4. 用户选择或输入照护者标签。
5. 用户点击加入。
6. 系统为该孩子创建成员关系。
7. 用户进入被邀请孩子的首页。

线框要求：

- 无效和过期状态分别呈现。
- 已加入用户进入首页，不创建重复成员关系。
- 只有显式确认加入后才创建成员关系。

## Wireframe Notes

1. 首页保持记录优先。第一屏应立即提供两个主要记录动作。
2. 记录管理保持次级。记录页用于纠错和浏览，不是主要日常工作流。
3. 孩子上下文保持可见。记错孩子是重要风险，因此表单和 Sheet 在保存或确认前必须显示当前孩子。
4. 使用明确保存边界。打开应用、Sheet 或表单都不能创建健康事实。
5. 保留首页出现后的两步排便快速路径：记录排便 -> 确认。
6. 生病记录允许用一个有用事实完成。照护者不应被迫填写临床时间线。
7. 对详情使用渐进披露。标签、备注、用药历史和日期范围编辑不应拖慢快速路径。
8. 使用日期级记录。每个孩子每天一条排便记录，每个孩子每天一条生病日记录。
9. 时间线保持可扫描。按日展示记录事实；省略空字段；压缩无记录日期。
10. 避免医疗解释。不要包含诊断、推荐、严重度评分、异常警报、用药建议或报告式框架。
11. 邀请保持低频。共享访问重要，但属于设置，不属于首页。
12. 照护者标签保持轻量。标签只帮助归属说明，不是角色或权限。
13. 空状态保持平静。不要暗示照护者没有记录就是失败。
14. 保存失败时保留未保存输入。用户可能正在压力场景中记录。
15. 避免 MVP 范围蔓延。不要为照片、成长相册、聊天、评论、动态、图表、提醒、AI 摘要、药品数据库、家庭角色、习惯追踪、教师端或医生端门户制作线框。
